2
执行一个动作，观察世界变化
动作 & 转移 ›

现在让智能体走一步，看看世界如何变化。

运行step2_move.py
import gymnasium as gym import minihack env = gym.make("MiniHack-Room-5x5-v0", observation_keys=("chars_crop", "blstats")) obs, info = env.reset(seed=42) def show(obs, label=""): print(f"\n{label}") for row in obs["chars_crop"]: print(" " + "".join(chr(c) if c > 32 else "·" for c in row)) bl = obs["blstats"] print(f" 位置: ({bl[0]}, {bl[1]})") show(obs, "【移动前】") # 动作 2 = 向南走 obs, reward, terminated, truncated, info = env.step(2) show(obs, "【向南走一步后】") print(f" 奖励: {reward}") env.close()

🔑 这就是 MDP 的"转移函数" T(s'|s,a)

env.step(action) 就是 T 的实现。你给一个动作 a，环境返回新状态 s'（通过观测 obs 体现）和奖励 r。

一句话理解 MDP：你在某个状态 → 选一个动作 → 到达新状态 + 获得奖励。重复这个循环。

env.step(action)=转移函数 T(s'|s,a) obs（观测字典）=新状态 s' 的观测 reward=奖励函数 R(s,a) terminated=是否到达终止状态
3
动作有哪些？— 动作空间 A
MDP · A ›

MDP 的第二个元素是动作空间 A。让我们看看智能体能做什么。

运行step3_actions.py
import gymnasium as gym import minihack from nle import nethack env = gym.make("MiniHack-Room-5x5-v0") obs, info = env.reset() print(f"动作空间大小: {env.action_space.n} 种动作\n") # MiniHack/NLE 的动作名称 ACTIONS = { 0: "↖ 左上", 1: "↑ 上", 2: "↗ 右上", 3: "← 左", 5: "→ 右", 6: "↙ 左下", 7: "↓ 下", 8: "↘ 右下", 4: "等待(不动)", } print("常见动作（移动为主）：") for idx, name in ACTIONS.items(): print(f" 动作 {idx} = {name}") print(f"\n其余动作 (9-{env.action_space.n-1}): 拾取、吃、攻击、施法...") print("在你的项目中，还会额外加入 QUERY 动作！") env.close()

📐 MDP 动作空间在你项目中的扩展

标准 MiniHack: A = {8方向移动 + 等待 + 拾取 + 吃 + ...}
你的项目: A = A标准 ∪ {QUERY}

QUERY 是一个新动作——花费一个时间步去查询知识库。和其他动作一样，它也会被 env.step() 处理。

4
blstats：智能体的"内部状态"
MDP · S ›

除了看到的地图，智能体还有内部属性——血量、饥饿度。这些就是状态空间 S 的一部分。

运行step4_blstats.py
import gymnasium as gym import minihack env = gym.make("MiniHack-Room-5x5-v0", observation_keys=("chars_crop", "blstats")) obs, info = env.reset(seed=42) bl = obs["blstats"] # blstats 中与你项目最相关的字段 IMPORTANT = { 0: ("x坐标", bl[0]), 1: ("y坐标", bl[1]), 10: ("HP (当前)", bl[10]), 11: ("HP (最大)", bl[11]), 20: ("游戏步数", bl[20]), 21: ("饥饿状态", bl[21]), # ← 重要！ } print("blstats 关键字段（共27维）：\n") for idx, (name, val) in IMPORTANT.items(): marker = " ← 你项目的'能量'来源！" if idx == 21 else "" print(f" [{idx:2d}] {name:12s} = {val}{marker}") print(f"\n饥饿状态含义：") print(f" 0=正常 1=饥饿 2=虚弱 3=晕倒 4=昏厥 5=饿死") print(f"\n这就是你项目中 E_t 的自然基础！") env.close()

⚡ 这就是能量 E_t 的来源

blstats[21] 是 NetHack 的饥饿状态。你的项目会在此基础上建立自己的能量系统：
• 每步消耗 c_step 能量
• 吃正确食物恢复能量
• E_t ≤ 0 → 死亡

这把 MDP 的状态空间 S 扩展成了 S̃ = (S, E_t)

5
MDP vs POMDP — 你看不到全部
核心概念 ›

到目前为止你看到的 chars_crop 只有 9×9。但完整地图是 21×79。来对比一下：

运行step5_pomdp.py
import gymnasium as gym import minihack env = gym.make("MiniHack-Room-15x15-v0", observation_keys=("chars", "chars_crop", "blstats")) obs, info = env.reset(seed=42) # ① 完整地图（上帝视角） print("① 完整地图 chars (21×79) — 上帝视角：") for row in obs["chars"]: line = "".join(chr(c) if c > 32 else " " for c in row).rstrip() if line.strip(): print(f" {line}") # ② 智能体的视野（局部裁剪） print("\n② 智能体看到的 chars_crop (9×9) — 只有这么多：") for row in obs["chars_crop"]: print(" " + "".join(chr(c) if c > 32 else "·" for c in row)) print("\n★ 关键区别：") print(" 完整地图 = MDP 的'状态 s'（包含一切信息）") print(" 裁剪视野 = POMDP 的'观测 o'（只能看到一部分）") print(" 观测 ≠ 状态 → 这就是为什么是 POMDP！") env.close()
🎯 一句话理解 POMDP：
MDP = 你能看到所有牌的牌局（完全信息）
POMDP = 你只能看到自己手牌的牌局（不完全信息）

在 MiniHack 中，用 chars 训练 ≈ MDP，用 chars_crop 训练 ≈ POMDP

🔍 所以 QUERY 的价值在哪里？

因为观测不完整，智能体不知道某些关键信息（哪种食物安全？前方有没有陷阱？）。
QUERY 动作 = 花一步时间去问知识库 → 获得 hint → 补充缺失信息。
这就是"主动减少不确定性"。

6
跑一个完整 Episode — 累积奖励
RL 目标 ›

RL 的目标是最大化累积奖励。让我们跑一局看看"好策略"和"差策略"的区别。

运行step6_episode.py
import gymnasium as gym import minihack env = gym.make("MiniHack-Room-5x5-v0", observation_keys=("chars_crop", "blstats"), max_episode_steps=30) total_reward = 0 obs, info = env.reset(seed=42) print("随机策略的一局游戏：") print(f"{'步':>3} {'动作':>4} {'奖励':>6} {'累积':>7}") print("-" * 28) for step in range(30): action = env.action_space.sample() # 随机策略 obs, reward, terminated, truncated, info = env.step(action) total_reward += reward done = terminated or truncated print(f"{step+1:3d} {action:4d} {reward:+6.2f} {total_reward:+7.3f}") if done: reason = "到达目标!" if reward > 0.5 else "超时" print(f"\n结果: {reason}") break print(f"\n总奖励 G = {total_reward:.3f}") print(f"RL的目标：找到策略π*让这个G尽可能大") env.close()

📈 G = r₁ + γr₂ + γ²r₃ + ...

每步 -0.01（鼓励快速完成），到达目标 +1.0。
随机策略的 G 通常很低（随便晃悠）。
好策略直奔目标，G ≈ +1.0。

RL 就是从随机策略出发，通过试错学习找到好策略。

7
能量门控 — 何时该 QUERY？
项目核心 ›

最后一步：理解你项目的核心创新——用能量状态决定何时查询。

运行step7_gating.py
import numpy as np def sigmoid(x): return 1.0 / (1.0 + np.exp(-x)) # 参数 E_star = 70 # 能量稳态设定点 alpha = 0.15 # 门控斜率（越大越像硬阈值） epsilon = 10 # 偏移量 # 模拟不同能量水平下的查询概率 print("能量 E_t → 查询概率 P(QUERY)") print("-" * 42) for E in [100, 90, 80, 70, 60, 50, 40, 30, 20, 10]: p = sigmoid(alpha * (E_star - E - epsilon)) bar = "█" * int(p * 30) + "░" * (30 - int(p * 30)) print(f" E={E:3d} P={p:.3f} {bar}") print(f""" 观察： 能量高(100) → P≈0.00 几乎不查询（没必要） 能量中(60) → P≈0.18 开始考虑查询 能量低(20) → P≈0.99 几乎一定查询（生存危机！） 这就是内稳态门控： 能量偏离稳态 → 触发信息获取 → 帮助决策 → 恢复稳态 就像人饿了会更主动找食物信息一样。""")

🧬 p(QUERY | E_t) = σ(α · (E* − E_t − ε))

这个公式就是你整个项目的核心：
• E* = 能量稳态点（"舒适区"）
• E_t = 当前能量
• 当 E_t 远低于 E* → σ输入正值大 → P(QUERY)↑
• 当 E_t 接近 E* → σ输入负值 → P(QUERY)↓

你项目的实验就是对比这个门控策略 vs 其他4种基线（不查/总查/定期/随机）。
